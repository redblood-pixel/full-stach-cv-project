# full-stach-cv-project
Fullstack приложение (Web UI + backend) для детекции игрушек по фотографии

## Описание проекта

Приложение для детекции игрушек на изображениях и видео с использованием модели YOLOv8. Проект представляет собой веб-интерфейс с backend-сервером на Flask, который обрабатывает загруженные файлы и возвращает результаты детекции.

## Структура проекта

```
full-stach-cv-project/
├── app.py                    # Основной файл приложения Flask
├── config.yaml              # Конфигурационный файл приложения
├── train_model/             # Директория для файлов обучения модели
├── src/                     # Исходный код приложения
│   ├── config/              # Конфигурационные модули
│   ├── services/            # Бизнес-логика приложения
│   └── utils/               # Вспомогательные утилиты

│   ├── css/                 # CSS стили
```

## Установка и настройка

### Предварительные требования

- Python 3.8+
- pip (менеджер пакетов Python)
- Установленная модель YOLOv8 (yolov8n.pt)

### Установка зависимостей

```bash
# Создание виртуального окружения (рекомендуется)
python -m venv venv

# Активация виртуального окружения
# На macOS/Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt
```

### Настройка конфигурации

Приложение настраивается через файл `config.yaml`. Основные параметры:

```yaml
app:
  host: 0.0.0.0      # Хост для запуска сервера
  port: 5001         # Порт для запуска сервера
  debug: true        # Режим отладки

model:
  path: yolov8n.pt   # Путь к файлу модели YOLOv8
  class_id: 77       # ID класса для детекции (77 - плюшевый медведь в COCO)
  class_name: "teddy bear"  # Название класса для отображения

paths:
  uploads: static/uploads           # Директория для загруженных файлов
  results: static/results           # Директория для результатов обработки
  reports: reports                  # Директория для отчетов
  history_file: static/request_history.json  # Файл для хранения истории запросов
```

## Запуск приложения

```bash
# Запуск сервера
python app.py

# Или с указанием другого хоста/порта из конфигурации
python app.py --host 0.0.0.0 --port 5001
```

После запуска приложение будет доступно по адресу `http://localhost:5001` (или другому порту, указанному в конфигурации).

## Использование

## Формат генерируемого отчета

Отчет включает в себя визуализацию данных о детекции игрушек, представленную в виде графика. Ключевой аспект графика - отслеживание динамики количества игрушек с течением времени. График показывает, как игрушки появляются и исчезают в кадре, позволяя анализировать их перемещение и поведение.

- По оси X отображается время (для видео) или последовательность изображений (для групп изображений)
- По оси Y отображается количество обнаруженных игрушек
- Линия графика показывает изменения в количестве игрушек, выявляя моменты их появления и исчезновения
- Пиковые значения указывают на максимальное количество одновременно видимых игрушек
- Снижение значений до нуля указывает на полное исчезновение всех игрушек из кадра

Эта динамическая визуализация позволяет понять не только общее количество обнаруженных игрушек, но и их поведение во времени, что особенно полезно для анализа видео-последовательностей.
